An anchor layout is typically used when making forms. Flowing types of layouts are generally not very useful since we're working with labels and form elements and not with chunks of text: we always want the labels to be positioned relative to the input fields, they are no part of a flowing text.

An anchor layout will not do that, but they will allow you to position form elements relative to the edges of the layout box. This way you'll achieve the same results.

A form will usually be rendered as a two column table with one column for the labels and another for the input fields. You don't want the label column to resize because it contains predefined texts. Resizing this column would mean a larger distance between the labels and the input fields and therefore cause an illegible form.

When resizing you want the extra space for the benefit of data entry. So only the input fields should be resized, and an anchor layout will do just that. You indicate which elements resize along with the form and which don't.

In this example we examine a very simple but also commonly used form: a popup box with just an input field and a text area. When resizing the form it is clear where we want use the extra space:

  1. the labels don't change their content so these should keep the same size,
  2. the input field does not need some extra height but it always could use some extra width,
  3. the text area obviously just can't be big enough in this case.

This basic example should give you an very idea of what an anchor layout is. Only by extending this sample you can create some very interesting dialogs. But the usage of anchor layout can be much more complex: think of multiple anchor layouts on a single form.
